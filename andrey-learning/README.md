repository — особый тип папки, хранилище со всеми возможными файлами и папками.

commit — новая версия файлов, отличная от предыдущих.

branch — независимые от основного потока изменения, для создания/корректировки новых фич, которые потом добавятся в основной поток.

staging area — промежуточное место для хранения изменений между текущим состоянием и коммитом.

merge — слияние разных веток.

pull request — запрос на слияние изменений из одной ветки в другую.

git init — создание папки .git в проекте для работы с Git.

git add — добавление изменений в staging area.

git commit — перенос изменений из staging area в коммит.

git push — отправка коммитов в текущую ветку на удалённый репозиторий.

git checkout — переключение между ветками.
