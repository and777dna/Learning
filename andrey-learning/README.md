repository — особый тип папки, хранилище со всеми возможными файлами и папками.

commit — новая версия файлов, отличная от предыдущих.

branch — независимые от основного потока изменения, для создания/корректировки новых фич, которые потом добавятся в основной поток.

staging area — промежуточное место для хранения изменений между текущим состоянием и коммитом.

merge — слияние разных веток.

pull request — запрос на слияние изменений из одной ветки в другую.

git init — создание папки .git в проекте для работы с Git.

git add — добавление изменений в staging area.

git commit — перенос изменений из staging area в коммит.

git push — отправка коммитов в текущую ветку на удалённый репозиторий.


git checkout — переключение между ветками.



Single responsibility SOLID-в каждом методе/классе должна быть только 1 ответственность,  которая полноценно расписана без ссылок.

Array-reference type в C#, имплементируется(!наследуется) от System namespace, Object. Array всегда фиксированного размера, чтоб добавить новый элемент нужно пересоздать ссылку. Доступ к элементам через индекс со сложностью O(1).

List Class-имплементирует несколько видов namespace, которые обеспечивают гибкий подход для работы с строго типизированными объектами  в List. Внутри List class при инициализации создается _array. Можем изменить размер List без пересоздания его вручную.

Dictionary-также как List class имплементирует namespaces, основан на key-value отношении для каждого элемента. 

Static-относится к классу, общий для всех объектов созданный этим классом. 
instance-для каждого объекта свое значение внутри экземпляра, для этого нужно создать с помощью слова new.

Модификаторы доступа:private, public, internal, protected

Public-может быть доступно вне данного namespace, в том числе в другой программе.
Private-может быть доступно только внутри данного класса и его наследников. 
internal-доступно только в рамка текущего namespace.

Чем field отличается от property?
field-переменная любого типа, объявленного в class.можем задать полю модификатор доступа.
property-производная от field с теми же характеристиками, а также возможностью менять, читать данные, а также ввести ограничения.

Зачем нужны методы? Метод=функция-часть кода, которая выполняется только когда мы её вызываем, может быть с параметрами. Нужна чтоб задавать поведение программе. 

Queue-реализует FIFO механизм, для конкретных задач, например для breadth first search.
Stack-реализует LIFO механизм, например необходим для backtracking, для inorder бинарного дерева чтоб узнать если дерево правильно построено. Если значения в stack будут упорядочены, то дерево правильное.
Linked List-датовый тип, когда у элемента есть свое значение, а также у него есть ссылка на следующий элемент.
