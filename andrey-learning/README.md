repository-особый тип папки, хранилище со всеми возможными файлами, папками
commit-новая версия файлов, отличная от предыдущих.
branch-независимые от основного потока изменения, для создания/корректировки новых фич, которые потом добавятся в основной поток.
staging area-промежуточное место для хранения изменений между текущим состоянием и комитом.
merge-слияние разных веток.
pull request-?
git init-создание папки .git в файле для работы с гитом.
git add-добавление изменений в staging area.
git commit-перенос изменений из staging area до комита.
git push-вложить комит до текущей ветки.
git checkout-переключиться между ветками.
